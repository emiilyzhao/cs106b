Q3. Does it take the same amount of work to compute `isPerfect` on the number 10 as it does on the number 1000? Why or why not? Does it take the same amount of work for `findPerfects` to search the range of numbers from 1-1000 as it does to search the numbers from 1000-2000? Why or why not?
A3. Computing isPerfect will take longer for 1000 than for 10, because isPerfect calls divisorSum,
    which iterates from 0 to n-1. divisorSum will take longer for a larger number.
    findPerfects will take longer for 1000-2000 than 1-1000, because it calls isPerfect for each number in the range.
    Since isPerfect takes longer for larger numbers, findPerfects will take longer to compute the higher range.

Soundex
-------

Q10. What is the Soundex code for "Angelou"? What is the code for your own surname?
A10. A524. The code for my surname "Zhao" is Z000.

Q11. Before writing any code, brainstorm your plan of attack and sketch how you might decompose the work into smaller tasks. Briefly describe your decomposition strategy.
A11. Remove all non-letter digits. Convert to upper-case and then encode all the letters.
     Remove duplicated digits. Change first digit to upper-case letter. Remove zero digits.
     Pad with 0s or truncate for a size of 4 digits.


Q12. Think about one or more examples of a class of names that the Soundex system might not work well for. Explain what this class of names is and why the system might incorrectly group them or mis-categorize one of the names.
A12. Names that are from non-English backgrounds, such as many Asian, African, or
     East European names, may have different pronounciations that differ from English.
     These names would be misrepresented by Soundex. For example, since Soundex generally
     encodes only consonant sounds, non-English names that rely on the different vowel
     sounds may be misrepresented as similar even when they are differently pronounced.

Q13. Suppose you are a software engineer working for the U.S. government. You have been tasked with implementing a system that collects names as part of the Census survey and analyzes these names using a phonetic algorithm. Your boss suggests using Soundex and notes that the algorithm is only expected to work for ASCII-encoded strings, since supporting Unicode would require extra work and time. What would your response be and why? What representational harms might result from building a system that exclusively uses Soundex and/or ASCII?
A13. I would respond that this would both result in misclassifications and an ethically
     flawed census. The US is composed of people with a diversity of names, and the census
     should accomodate for this. Additionally, since ASCII cannot represent alternative
     characters, the only way to add such names to the database would be to corrupt the
     data by removing characters, which would result in misclassifications. I would also
     propose that if the algorithm is only expected to work on ASCII, we can develop an
     alternative way to losslessly transcibe and phonetically match names in the context
     of the language in which the name comes from.

     A system that exclusively uses Soundex/ASCII creates representational harms like
     assimilation as people's names are forced to fit within English contexts, or
     unequal representation as more minority non-english or immigrant communities have
     increased errors in the census records.
